# SVHunter 使用说明

本说明适用于克隆本仓库后从零开始配置环境并使用 SVHunter Web 界面进行结构变异检测。

---

## 一、系统要求

| 组件 | 要求 |
|------|------|
| 操作系统 | Linux / macOS / Windows（WSL2） |
| Python | 3.11 |
| Node.js | >= 18 |
| conda | Miniconda 或 Anaconda |
| GPU（可选） | NVIDIA GPU，驱动支持 CUDA 11.8 |

> **WSL2 用户**：建议将内存上限调高，否则推理时可能导致 WSL 崩溃。
> 在 `C:\Users\<用户名>\.wslconfig` 中添加：
> ```ini
> [wsl2]
> memory=16GB
> swap=8GB
> ```
> 然后在 PowerShell 中执行 `wsl --shutdown` 并重新打开 WSL。

---

## 二、克隆仓库

```bash
git clone <仓库地址>
cd SVHunter-1
```

---

## 三、配置推理环境（svhunter-infer）

此环境用于运行 SVHunter 推理核心（`SVHunter.py`）。

```bash
conda create -n svhunter-infer python=3.11 -y
conda activate svhunter-infer
python -m pip install --upgrade pip setuptools wheel
python -m pip install "numpy==1.24.3" "pandas==2.0.3" "scipy==1.10.1" "scikit-learn==1.5.1" "pysam==0.22.1" "tensorflow==2.12.1"
```

### GPU 加速（可选，推荐）

如果你有 NVIDIA GPU，继续安装以下 CUDA 库：

```bash
python -m pip install nvidia-cudnn-cu11==8.6.0.163 nvidia-cuda-runtime-cu11==11.8.89 nvidia-cufft-cu11==10.9.0.58 nvidia-curand-cu11==10.3.0.86 nvidia-cusolver-cu11==11.4.1.48 nvidia-cusparse-cu11==11.7.5.86
```

### 验证推理环境

```bash
python -c "import tensorflow as tf, numpy, pandas, pysam, sklearn, scipy; print('tf', tf.__version__, '| numpy', numpy.__version__)"
```

应输出：`tf 2.12.1 | numpy 1.24.3`

## 四、配置后端环境（svhunter-api）

此环境用于运行 Web 界面的后端 API 服务。

```bash
conda create -n svhunter-api python=3.11 -y
conda run -n svhunter-api python -m pip install -r version4/server/requirements.txt
```

### 验证后端环境

```bash
conda run -n svhunter-api python -c "import fastapi, uvicorn, pydantic; print('ok')"
```

## 六、修改配置（重要）

打开 `version4/server/services/runner.py`，找到以下两行，改为你自己的路径：

```python
# 推理 Python 解释器路径
SVHUNTER_PYTHON = "/home/<你的用户名>/miniconda3/envs/svhunter-infer/bin/python"

# CUDA 库路径（如不使用 GPU 可忽略）
_NVIDIA_BASE = "/home/<你的用户名>/miniconda3/envs/svhunter-infer/lib/python3.11/site-packages/nvidia"
```

找到你的 Python 路径：

```bash
conda activate svhunter-infer
which python
```

## 八、启动服务

需要两个终端窗口。

**终端 1 — 启动后端：**

```bash
cd version4
/home/<你的用户名>/miniconda3/envs/svhunter-api/bin/python -m uvicorn server.main:app --port 8000
```

验证后端正常：打开 http://localhost:8000/api/health，应返回 `{"status":"ok"}`。

**终端 2 — 启动前端：**

```bash
cd version4
npm run dev
```

打开浏览器访问 **http://localhost:5173**

## 九、使用界面运行推理

### 第一步：生成特征数据（Pipeline → Generate）

填写以下参数后点击「开始生成特征数据」：

| 字段 | 说明 | 示例 |
|------|------|------|
| BAM 文件路径 | 长读长比对文件的**绝对路径** | `/data/HG002.bam` |
| 输出目录 | 特征数据输出目录的**绝对路径** | `/data/SVHunter-1/datapath` |
| 线程数 | 并行线程数，建议 1-4 | `1` |
| 染色体 | 逗号分隔，留空=全部 | `1,2` 或留空 |

> ⚠️ 染色体名称必须与 BAM 文件一致。大多数 GRCh38 BAM 文件使用 `1`、`2`，而非 `chr1`、`chr2`。

进度日志会实时显示在右侧面板。

### 第二步：检测 SV（Pipeline → Call）

填写以下参数后点击「开始 SV 检测」：

| 字段 | 说明 |
|------|------|
| 模型权重路径 | `model_predict.h5` 的绝对路径 |
| 特征数据目录 | 第一步的输出目录 |
| BAM 文件路径 | 与第一步相同 |
| 预测输出目录 | `predict/` 的绝对路径 |
| VCF 输出目录 | `vcf/` 的绝对路径 |
| 线程数 | 建议 1-4 |
| 染色体 | 与第一步相同 |
| GPU 数量 | 有 GPU 填 `1`，纯 CPU 填 `0` |

### 第三步：查看结果（Results 页面）

推理完成后切换到 Results 页面，结果会自动从 `vcf/` 目录加载，展示：

- SV 总数及按类型统计（DEL / INS / DUP / INV / TRA）
- 按染色体分布图
- 可搜索的 VCF 记录表格

## 十、常见问题

**Q：染色体填了但提示找不到数据？**
A：检查 BAM 文件里的染色体命名。用以下命令查看：
```bash
conda run -n svhunter-infer python -c "import pysam; b=pysam.AlignmentFile('/path/to/your.bam'); print([r.contig for r in b.get_index_statistics()[:5]])"
```

**Q：GPU 没有被使用？**
A：确认已安装 CUDA 库（第三节 GPU 加速部分）。如果有多张 GPU，检查 `SVHunter.py` 第 13 行和 `SVHunter_detect.py` 第 29 行的 `CUDA_VISIBLE_DEVICES` 设置是否与你的 GPU 编号匹配（通常为 `"0"`）。

**Q：WSL 在推理时崩溃？**
A：内存不足。参考第一节配置 `.wslconfig` 将内存上限设为 16GB 或以上。

**Q：Results 页面显示演示数据而非真实结果？**
A：后端未连接或 VCF 目录不在默认位置。确认后端已启动，且 `vcf/` 目录在仓库根目录下。也可在 Results 页面的输入框手动填写 VCF 目录路径。
